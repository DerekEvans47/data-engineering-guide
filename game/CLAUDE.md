# Claude Code — Game Project Rules

Standalone tower-defense game (*Quiz Defense*). Pure HTML/CSS/vanilla JS, no build step.
This repo was split out of the `data-engineering-guide` monorepo; it shares no code with
the learning material. When this folder becomes the root of its own repository, everything
below applies at the repo root.

---

## Commit & PR Hygiene — No Personal Identifiers

**ABSOLUTE RULE — zero exceptions:**

Never include Claude session URLs, chat links, or any personally identifiable information in:
- Commit messages (subject or body)
- PR titles or descriptions (any section, any footer)
- Changelog / release notes
- Any file that gets committed or pushed

**Specifically banned:**
- `claude.ai/code/session_*` URLs — in any form, anywhere
- `Claude-Session:` trailers
- `_Generated by [Claude Code](...)_` / `_Generated with [Claude Code](...)_` footers
- Any `claude.ai` link
- Internal task IDs or session identifiers

**Only allowed attribution:** `Co-Authored-By: Claude <noreply@anthropic.com>`

> **Why:** session URLs are personal identifiers tied to the owner's account. This is
> enforced by the git hooks below, but the hooks are a backstop — self-check every commit
> message and PR body before submitting. GitHub API calls (PR bodies) are **not** covered
> by any hook.

## Git hooks — setup in this repo (and how they got here)

The PII enforcement and the browser verifier run from two git hooks. Git does **not** clone
`.git/hooks/`, so the hook *scripts* are tracked in the repo and re-installed automatically:

- **Sources (tracked):** `.claude/skills/pre-commit-hook.sh`, `.claude/skills/pre-push-hook.sh`
- **Auto-install:** `.claude/settings.json` has a `SessionStart` hook that copies both
  scripts into `.git/hooks/` and `chmod +x`es them at the start of every Claude Code
  session. Open the repo once in Claude Code and the hooks are live.

**What they enforce:**
- `pre-commit` — scans **staged file content** for session URLs / identifiers and blocks
  the commit if any are found.
- `pre-push` — blocks any pushed **commit message/body** containing a session URL, a
  `Claude-Session:` trailer, or a generated-by footer; and runs `verify.sh` when app
  source files (`*.js`/`*.css`/`*.html`/`*.json`/`assets/`) are in the push.

**Manual install** (if not using Claude Code, or to install right now):

```bash
cp .claude/skills/pre-commit-hook.sh .git/hooks/pre-commit && chmod +x .git/hooks/pre-commit
cp .claude/skills/pre-push-hook.sh  .git/hooks/pre-push   && chmod +x .git/hooks/pre-push
```

To verify they're active: `ls -l .git/hooks/pre-commit .git/hooks/pre-push` (both should be
executable), then make a test commit whose message contains `claude.ai/code/session_test` —
the push should be rejected.

## Service Worker Checklist

Any time you change a file in this repo, bump the cache version in `sw.js` before committing:

```bash
grep "const CACHE" sw.js
# const CACHE = 'quiz-defense-game-vN';  → bump N by 1
```

Skipping this leaves browsers serving stale cached files after deploy. `APP_VERSION` (the
home-screen version badge, in `drill-core.js`) auto-derives from the active
`quiz-defense-game-v*` cache name — do **not** hardcode it.

## Verification Gate — before every push touching source

```bash
bash verify.sh
```

Drives Chromium via Playwright through the full flow (home → region map → battle → wave
start) and asserts the game runs standalone with no quiz gating. **Exit 0 = safe to push.**
The pre-push hook runs this automatically when source files change. Known non-fatal noise:
`ERR_FAILED` on `fonts.googleapis.com` (no network in sandboxes).

## Creator Mode & tuning

- **Creator Mode** — `?dev=1` on the battle screen shows the map editor + tuning/relic
  editor chips, and grants a 99999 testing purse (real start gold is 160).
- **`config.json`** — ALL tower/enemy/power-up/relic tuning plus the Frontier Town knobs.
  Balance changes are JSON edits, not JS edits. The in-app relic editor's 📋 export pastes
  the full updated `config.json`.

## Branch naming

`<type>/<short-hyphenated-description>` (≤5 words), e.g. `fix/wave-preview-flicker`,
`feat/game-speed-toggle`. Never use auto-generated slugs as the final branch name.
