SCD Type 2 Pattern (Spark SQL / MERGE)

-- Upsert with history preservation
MERGE INTO silver.dim_customer AS target
USING (
  SELECT *, TRUE AS is_new FROM staging.customer_updates
) AS source
ON target.customer_id = source.customer_id AND target.is_current = TRUE
WHEN MATCHED AND (target.name != source.name OR target.address != source.address) THEN
  UPDATE SET is_current = FALSE, valid_to = CURRENT_TIMESTAMP
WHEN NOT MATCHED THEN
  INSERT (customer_id, name, address, is_current, valid_from, valid_to)
  VALUES (source.customer_id, source.name, source.address, TRUE, CURRENT_TIMESTAMP, NULL);

Python — Data Engineering Patterns

Polars — Fast DataFrame Pattern

import polars as pl

df = pl.read_parquet("trips.parquet")

result = (
    df
    .filter(pl.col("fare_amount") > 0)
    .filter(pl.col("trip_distance").is_between(0.1, 200))
    .with_columns([
        pl.col("pickup_datetime").dt.hour().alias("pickup_hour"),
        (pl.col("tip_amount") / pl.col("fare_amount")).alias("tip_pct"),
    ])
    .group_by(["pickup_hour", "payment_type"])
    .agg([
        pl.col("fare_amount").mean().alias("avg_fare"),
        pl.col("tip_pct").median().alias("median_tip_pct"),
        pl.len().alias("trip_count"),
    ])
    .sort("pickup_hour")
)

Azure Blob Storage — Upload / Download

from azure.storage.blob import BlobServiceClient
from pathlib import Path

client = BlobServiceClient.from_connection_string(os.environ["AZURE_STORAGE_CONN_STR"])

# Upload
def upload_parquet(local_path: Path, container: str, blob_name: str) -> None:
    blob = client.get_blob_client(container, blob_name)
    blob.upload_blob(local_path.read_bytes(), overwrite=True)

# Download
def download_parquet(container: str, blob_name: str, local_path: Path) -> None:
    blob = client.get_blob_client(container, blob_name)
    local_path.write_bytes(blob.download_blob().readall())

Retry Decorator for API Calls

import time
import functools

def retry(max_attempts=3, delay=1.0, backoff=2.0):
    def decorator(fn):
        @functools.wraps(fn)
        def wrapper(*args, **kwargs):
            attempt, wait = 0, delay
            while attempt < max_attempts:
                try:
                    return fn(*args, **kwargs)
                except Exception as e:
                    attempt += 1
                    if attempt >= max_attempts:
                        raise
                    time.sleep(wait)
                    wait *= backoff
        return wrapper
    return decorator

@retry(max_attempts=3, delay=2.0)
def call_external_api(url: str) -> dict:
    # ... request logic
    pass

PySpark Quick Reference

Figure B.2. PySpark narrow vs wide transformations. Narrow transformations are cheap and freely chainable; wide transformations trigger network shuffles. Performance tuning almost always starts with minimizing shuffles.

Key PySpark Patterns

from pyspark.sql import functions as F
from pyspark.sql.window import Window

# Broadcast join (avoid shuffle for small tables)
from pyspark.sql.functions import broadcast
result = large_df.join(broadcast(small_df), "key")

# Window function
w = Window.partitionBy("region").orderBy("trip_date")
df = df.withColumn("running_total", F.sum("revenue").over(w))

# Conditional column
df = df.withColumn("tier",
    F.when(F.col("revenue") > 1000, "high")
     .when(F.col("revenue") > 500, "mid")
     .otherwise("low"))

# Explode array column
df = df.withColumn("tag", F.explode(F.col("tags")))

# Pivot
df.groupBy("region").pivot("year").agg(F.sum("revenue"))

Git — Data Engineering Workflow Commands

# Daily workflow
git checkout -b feature/add-trip-model    # New branch per feature
git add -p                                 # Stage hunks interactively (preferred over git add .)
git commit -m "feat(trips): add fct_trips incremental model"
git push origin feature/add-trip-model
# Open PR → CI runs dbt test → merge to main

# Undo staged changes (not committed)
git restore --staged models/fct_trips.sql

# Amend last commit message (before push)
git commit --amend -m "new message"

# Interactive rebase to squash commits
git rebase -i HEAD~3

# Stash changes and apply later
git stash push -m "WIP: trip model"
git stash pop

# See who changed a line (blame)
git log -L 12,15:models/fct_trips.sql

# Pull with rebase (preferred for DE repos — cleaner history)
git pull --rebase origin main

Window vs GROUP BY

PARTITION BY in a window function groups rows like GROUP BY — but doesn't collapse them. Every source row appears in the output. Use it whenever you need an aggregation alongside the original row detail.

Filter Before Join

In PySpark, always apply .filter() before .join(). Filtering reduces the rows that must be shuffled. This is the single most impactful query optimization available without changing the query logic.

dbt full-refresh

Running dbt run --full-refresh on an incremental model drops and recreates the table. On large tables in production, this can take hours. Always test incremental logic before relying on it — and add --target dev when experimenting.
